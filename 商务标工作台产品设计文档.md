# 商务标工作台产品设计文档

## 1. 产品目标

新增“商务标工作台”，提供独立于技术标的完整业务流程，实现从招标文件解析、要素提取、目录生成、数据填充、AI内容撰写到最终 Docx 文件导出的全链路闭环，以提升商务标书编制的效率和准确性。

## 2. 核心需求

### 2.1 独立的业务流程与模型配置
- **流程隔离**：商务标工作台应拥有独立的操作入口、项目列表和数据流转状态，不与现有技术标工作台混用。
- **专属 Prompt**：针对商务标的专业性（如报价、资质、法务合规等），配置专属的 AI 提示词，确保生成的文本符合商务投标的语言规范和逻辑严谨性。

### 2.2 智能解析与要素提取
- **文档解析**：支持上传并解析招标文件，智能提取关键商务要素（如招标人、项目名称、投标保证金、资质要求、工期等）。
- **交互方式**：
  - 采用**标签页（Tab）导航**展示不同类别的要素提取结果，便于分类查看。
  - 支持**原文高亮对应**：点击提取出的要素时，自动定位并高亮显示招标文件中的对应原文，方便用户核对信息的准确性。

### 2.3 智能目录生成
- **生成依据**：基于招标文件中的“投标文件格式”或“投标文件组成”相关章节内容，AI 自动生成对应的商务标目录大纲。
- **目录编辑**：支持用户对生成的目录结构进行手动增删改查及层级调整。

### 2.4 数据填充与管理
- **系统数据联动**：与现有的资源管理数据库（公司信息、资质证明、财务信息、人员业绩等）打通，根据目录节点的要求，自动匹配并填充相应的结构化数据。
- **人工干预与附件选择**：支持用户手动修改自动填充的数据，并提供可视化的图片/附件选择器，便于从系统资源库中插入相关证明材料。

### 2.5 AI辅助内容撰写
- **智能续写/生成**：针对需要文字阐述的章节（如项目承诺书、授权委托书等），调用大语言模型进行内容生成。
- **语言风格**：确保生成的文本符合商务文件严谨、专业的语言风格。

### 2.6 格式化文档导出
- **导出格式**：支持将最终完成的商务标一键导出为 `.docx` 格式。
- **排版规范**：导出的文档需严格遵循预设的排版格式要求（如字体、字号、段落间距、页眉页脚等），确保输出文件的规范性。

  - 提供导出 Docx 按钮，可配置导出样式和排版选项。

## 4. 后续技术实现规划建议

- **前端架构**：复用现有的技术标工作台部分 UI 组件（如 PDF 预览器、树状组件、富文本编辑器），调整状态管理以支持商务标特有的数据结构。
- **后端架构**：增加商务标专用的 API 路由（如 `/api/business-bids`）和数据库模型（如 `BusinessBidProject`, `BusinessBidDirectory` 等），配置专属的 LLM Prompt 链。
- **数据集成**：确保新工作台与现有的企业资源库模块（公司信息、资质、业绩等）平滑对接，实现数据的无缝调用。
