# 商务标工作台开发指导文档

本文档基于《商务标工作台产品设计文档》，为开发团队提供“商务标工作台”从 0 到 1 的技术实现指导、架构设计与阶段性开发计划。

---

## 1. 架构与技术栈约束

1. **前端技术栈**：React (Hooks), TypeScript, Tailwind CSS, 状态管理 (Zustand/Context API，推荐针对复杂文档树使用 Zustand), 路由 (React Router)。
2. **后端技术栈**：FastAPI (Python), SQLAlchemy (MySQL ORM), MinIO (对象存储), 异步处理 (Asyncio)。
3. **AI 大模型接入**：复用现有的 LLM 接入层（OpenAI 兼容接口），但需针对商务标配置**独立的 Prompt 模板**与解析流水线。
4. **文档导出**：使用 `python-docx` 或类似库生成标准 `.docx` 文件，结合预设的 Word 模板进行占位符替换与样式控制。

---

## 2. 数据库设计指南

为实现商务标业务流程的隔离，需在数据库中新增以下核心表模型：

### 2.1 `BusinessBidProject` (商务标项目表)
- `id`: UUID / 主键
- `project_name`: 商务标项目名称
- `status`: 状态 (新建、解析中、编写中、已完成等)
- `tender_document_url`: 原始招标文件存储路径 (MinIO)
- `created_at`, `updated_at`: 时间戳

### 2.2 `BusinessBidElement` (商务标要素提取表)
- `id`: 主键
- `project_id`: 关联的商务标项目 ID
- `category`: 要素类别 (如：基本信息、资质要求、财务要求、人员要求)
- `key_name`: 要素键名 (如：招标人、工期、投标保证金)
- `extracted_value`: 提取的文本值
- `source_text`: 招标文件中的原文片段
- `page_number`: 所在的 PDF 页码 (用于前端高亮定位)
- `bounding_box`: 坐标信息 (用于精准高亮)

### 2.3 `BusinessBidDirectory` (商务标目录结构表)
- `id`: 主键
- `project_id`: 关联的商务标项目 ID
- `parent_id`: 父节点 ID (实现树状结构，根节点为空)
- `title`: 目录标题 (如：一、法定代表人身份证明)
- `level`: 层级 (1, 2, 3...)
- `sort_order`: 排序序号
- `content_type`: 内容类型 (纯文本、富文本、表单数据、图片附件、自动生成)
- `content`: 实际内容或关联的数据源配置

## 3. 后端 API 接口设计

新增路由前缀：`/api/business-bids`

- **项目管理 API**
  - `GET /`：获取商务标项目列表 (支持分页、筛选)。
  - `POST /`：创建商务标项目。
  - `GET /{id}`：获取单个项目详情。
  
- **文档解析与要素提取 API**
  - `POST /{id}/upload-tender`：上传招标文件并触发异步解析任务。
  - `GET /{id}/elements`：获取已提取的要素列表 (按分类分组返回，附带原文坐标数据)。

- **目录生成与管理 API**
  - `POST /{id}/generate-directory`：调用 AI 生成初始目录大纲。
  - `GET /{id}/directories`：获取项目的完整目录树。
  - `PUT /{id}/directories`：批量更新目录树 (支持拖拽排序后的保存)。
  - `POST /{id}/directories/{node_id}/content`：保存/更新特定节点的内容。

- **AI 写作与导出 API**
  - `POST /{id}/generate-content`：传入节点上下文与 Prompt，流式 (SSE) 返回 AI 生成的商务文本。
  - `POST /{id}/export-docx`：根据当前目录结构和内容，合成并导出 Docx 文件。

## 4. 前端开发指南

### 4.1 核心组件复用与封装
1. **PDF 预览与高亮组件**：复用现有的 PDF 查看器，但需扩展**文本/坐标高亮**能力。当用户点击右侧“提取要素”时，触发 PDF 组件滚动到指定页码并绘制高亮框。
2. **树状目录组件 (Tree)**：支持拖拽排序 (Drag & Drop)、新增节点、重命名、删除。
3. **富文本编辑器/表单注入器**：支持在富文本中插入“占位符变量”（如 `{{公司名称}}`），并支持一键从资源库选取图片（如营业执照）插入光标位置。

### 4.2 状态管理策略
商务标工作台的数据流较为复杂，建议将状态拆分为：
- `ProjectState`：当前项目基本信息。
- `ElementsState`：提取的要素数据（按 Tab 分类）。
- `DirectoryTreeState`：目录树结构（扁平化存储，选择器转换为树形结构，方便拖拽与更新）。
- `CurrentNodeState`：当前选中的目录节点及其对应的表单/内容数据。

## 5. AI 与大模型集成规范

### 5.1 商务标专属 Prompt 工程
1. **要素提取 Prompt**：
   - 输入：招标文件 OCR 解析后的文本片段。
   - 要求：提取出标准 JSON 格式的要素，必须包含 `category`, `key_name`, `extracted_value`, 并且要求 AI 返回提取该值所依赖的原始句子 `source_text`。
2. **目录生成 Prompt**：
   - 输入：招标文件中的“投标文件组成”章节。
   - 要求：生成层级清晰的 JSON 树状结构，剔除属于“技术标”的部分，保留“商务标”与“资信标”部分。
3. **内容生成 Prompt**：
   - 必须设定角色为人设：“你是一位资深且严谨的招投标专员。”
   - 语气要求：专业、正式、无感情色彩的公文语态。

## 6. 阶段实施计划 (Milestones)

- **Phase 1: 基础设施与项目管理 (Week 1)**
  - 完成数据库表结构变更及 Alembic 迁移。
  - 实现商务标项目的 CRUD 接口。
  - 前端搭建商务标项目列表页、创建弹窗及基础路由。
- **Phase 2: 解析与要素提取 (Week 2)**
  - 后端接入文件上传、OCR/PDF 解析逻辑，调试要素提取 Prompt。
  - 前端实现左右分栏布局：左侧 PDF 预览，右侧 Tab 标签页展示要素。
  - 实现点击要素联动左侧 PDF 高亮的功能（核心技术难点）。
- **Phase 3: 目录生成与管理 (Week 3)**
  - 后端实现 AI 目录生成接口及目录树的 CRUD。
  - 前端接入树状组件，实现拖拽排序和节点管理交互。
- **Phase 4: 数据填充与智能撰写 (Week 4)**
  - 打通现有的公司信息、资质证明等资源库数据源。
  - 前端实现目录节点选中后的右侧表单/富文本编辑区。
  - 接入 AI 智能写作 API（流式输出）。
- **Phase 5: Docx 导出与联调测试 (Week 5)**
  - 后端实现 `python-docx` 模板引擎，将目录与内容拼装并生成带样式的 Word 文档。
  - 前后端全链路联调，处理各种边界情况及 UI 细节打磨。
