### Docker Compose配置文件

**文件位置**: `D:/Temp/bsdoc-sw/hxybs-js/docker-compose.yml`

```yaml
version: '3.8'

services:
  # MySQL数据库服务
  mysql:
    image: mysql:8.0
    container_name: bsdoc-mysql
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: hxybs
      MYSQL_USER: hxybs
      MYSQL_PASSWORD: hxybs
      TZ: Asia/Shanghai
    ports:
      - "13306:3306"  # 使用13306端口避免Windows端口权限问题
    volumes:
      - mysql_data:/var/lib/mysql
      - ./docker/mysql/init:/docker-entrypoint-initdb.d  # 初始化SQL脚本
    command:
      - --character-set-server=utf8mb4
      - --collation-server=utf8mb4_unicode_ci
      - --default-time-zone='+08:00'
    networks:
      - bsdoc-network
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-uroot", "-proot"]
      interval: 10s
      timeout: 5s
      retries: 5

  # Redis服务（可选，用于缓存）
  redis:
    image: redis:7-alpine
    container_name: bsdoc-redis
    restart: unless-stopped
    ports:
      - "6380:6379"  # 使用6380端口避免冲突
    volumes:
      - redis_data:/data
    networks:
      - bsdoc-network
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  # MinIO对象存储（可选，用于文件存储）
  minio:
    image: minio/minio:latest
    container_name: bsdoc-minio
    restart: unless-stopped
    environment:
      MINIO_ROOT_USER: minioadmin
      MINIO_ROOT_PASSWORD: minioadmin
      TZ: Asia/Shanghai
    ports:
      - "9001:9000"   # API端口
      - "9002:9001"   # 控制台端口
    volumes:
      - minio_data:/data
    command: server /data --console-address ":9001"
    networks:
      - bsdoc-network
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:9000/minio/health/live"]
      interval: 10s
      timeout: 5s
      retries: 5

volumes:
  mysql_data:
    driver: local
  redis_data:
    driver: local
  minio_data:
    driver: local

networks:
  bsdoc-network:
    driver: bridge
```

## 故障排查

### 问题1: Docker端口绑定失败

**错误信息**: `Error response from daemon: ports are not available`

**解决方案**:

1. 检查端口占用情况:
```bash
netstat -an | findstr "13306"
```

2. 如果端口被占用，修改 `docker-compose.yml` 中的端口映射

### 问题2: MySQL连接失败

**错误信息**: `Can't connect to MySQL server on 'localhost:13306'`

**解决方案**:

1. 检查容器状态:
```bash
docker ps -a | findstr bsdoc-mysql
```

2. 查看容器日志:
```bash
docker logs bsdoc-mysql
```

3. 等待MySQL完全启动（可能需要30秒以上）

4. 检查健康状态:
```bash
docker inspect bsdoc-mysql | findstr Health
```

### 问题3: conda环境无法激活

**解决方案**:

1. 初始化conda（如果尚未初始化）:
```bash
conda init bash
# 或Windows PowerShell
conda init powershell
```

2. 重启终端后重新激活

### 问题4: Python包导入错误

**解决方案**:

1. 确认已激活正确的conda环境:
```bash
conda activate bsdoc
python -c "import fastapi; print(fastapi.__version__)"
```

2. 如果包缺失，重新安装:
```bash
pip install -r D:/Temp/bsdoc-sw/hxybs-js/backend/requirements.txt
```

### 问题5: 数据库表不存在

**解决方案**:

1. 运行数据库初始化:
```bash
cd D:/Temp/bsdoc-sw/hxybs-js/backend
python -c "from app.database import init_db; init_db()"
```

2. 或在应用启动时自动创建表
