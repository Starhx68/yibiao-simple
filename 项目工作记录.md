# 项目工作记录

## 2026-03-18

### 资料库上传图片被覆盖问题排查与修复
- 问题现象：在资料库页面上传一张图片后，再上传其他图片时，前一张可能被后一张覆盖。
- 根因：上传图片后会触发 OCR 智能填充，后端返回结果包含固定 URL 字段，前端直接合并结果导致已上传字段被覆盖。
- 处理结果：在以下页面的 smartFill 合并逻辑中，先过滤上传 URL 字段，再回填文本字段：
  - `frontend/src/pages/resource/CompanyInfoPage.tsx`
  - `frontend/src/pages/resource/QualificationManagement.tsx`
  - `frontend/src/pages/resource/PersonnelManagement.tsx`
  - `frontend/src/pages/resource/FinancialInfoManagement.tsx`
  - `frontend/src/pages/resource/PerformanceManagement.tsx`
- 验证：前端 `npm run build` 通过，诊断无新增报错。

### 业绩保存失败问题修复
- 问题现象：新增业绩保存失败，数据库报 `Data too long for column 'completion_status'`。
- 根因：`performances` 表中 `completion_status` 与 `acceptance_status` 字段长度不足。
- 处理结果：
  - 将模型字段由 `String(50)` 调整为 `Text`：
    - `backend/app/models/models.py`
  - 在数据库初始化/校正逻辑中加入列类型修正：
    - `backend/app/database.py`

### MinIO 图片上传稳定性增强
- 问题现状：`upload_image_to_server` 原实现缺少连接健康检查、失败重试和超时控制。
- 处理结果（`backend/app/services/file_service.py`）：
  - 增加 MinIO 连接健康检查（`bucket_exists` 异常分支明确报错）。
  - 增加重试机制（默认重试 3 次，指数退避，最大 5 秒）。
  - 增加超时控制（`asyncio.wait_for`，默认 20 秒）。
  - 失败信息包含重试次数与最后一次错误，便于定位。
- 新增配置项（`backend/app/config.py`）：
  - `minio_upload_max_retries: int = 3`
  - `minio_upload_timeout_seconds: float = 20`
- 推荐环境变量：
  - `MINIO_UPLOAD_MAX_RETRIES=3`
  - `MINIO_UPLOAD_TIMEOUT_SECONDS=20`
- 验证：
  - `python -m py_compile app/config.py app/services/file_service.py` 通过。
  - 诊断无新增报错。
